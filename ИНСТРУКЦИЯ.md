# Инструкция по запуску приложения

## Быстрый старт

### Шаг 1: Соберите фронтенд
```bash
cd Client
npm install
npm run build
```

### Шаг 2: Запустите сервер
```bash
cd ../Server
dotnet run
```

### Шаг 3: Откройте браузер
После запуска вы увидите сообщение с URL, например:
- `https://localhost:7064` или
- `http://localhost:5058`

Откройте этот URL в браузере.

---

## Структура проекта

Проект разделен на две папки:
- **Client/** - Vue 3 фронтенд
- **Server/** - .NET 10 бэкенд

---

## Если нужно пересобрать фронтенд

Если вы изменили файлы Vue и нужно пересобрать:

```bash
cd Client
npm run build
```

Собранные файлы автоматически попадут в `Server/wwwroot/`.

Затем снова запустите сервер:
```bash
cd ../Server
dotnet run
```

---

## Режим разработки (с hot-reload)

Если хотите разрабатывать с автоматической перезагрузкой:

### Терминал 1 - Backend:
```bash
cd Server
dotnet run
```

### Терминал 2 - Frontend (Vite dev server):
```bash
cd Client
npm run dev
```

Затем откройте `http://localhost:5173` в браузере.

---

## Проверка работы

1. Откройте приложение в браузере
2. Заполните форму на вкладке "Submit Form"
3. Нажмите "Submit Form"
4. Перейдите на вкладку "View Submissions"
5. Вы увидите вашу отправку в списке
6. Попробуйте поиск по любому полю

## Возможные проблемы

### Порт уже занят
Если порт занят, измените его в `Server/Properties/launchSettings.json`

### Ошибка при запуске
Убедитесь, что установлены:
- .NET 10 SDK
- Node.js 18+ (для сборки фронтенда)

### Фронтенд не отображается
Убедитесь, что папка `Server/wwwroot` содержит файлы:
- `index.html`
- `assets/index.js`
- `assets/index.css`

Если их нет, выполните:
```bash
cd Client
npm run build
```

### Процесс блокирует файл
Если видите ошибку "файл используется другим процессом":
1. Закройте все запущенные экземпляры приложения
2. Или перезапустите компьютер
3. Или используйте команду для остановки процесса:
   ```powershell
   taskkill /F /IM FormSubmissionApi.exe
   ```
