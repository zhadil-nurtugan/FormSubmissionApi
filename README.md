# Form Submission Application

A full-stack web application for submitting forms, storing submissions, and listing/searching them. Built with .NET 10 and Vue 3.

## Features

- **Dynamic Form Submission**: Submit forms with various field types (text, dropdown, date, radio, checkbox)
- **Form Validation**: Client-side validation for all form fields
- **Dynamic Backend**: REST API that accepts any form structure without hardcoded models
- **In-Memory Storage**: No database setup required - uses in-memory storage
- **Search & List**: View and search all submissions from different forms
- **Modern UI**: Beautiful, responsive interface built with Vue 3
- **Build System**: Vite bundles all JS and CSS into optimized files

## Architecture

### Backend (.NET 10)
- **ASP.NET Core Web API** with REST endpoints
- **Dynamic Data Handling**: Uses `Dictionary<string, object>` to accept any form structure
- **In-Memory Storage**: `ConcurrentDictionary` for thread-safe storage
- **Service Layer**: Clean separation with `ISubmissionService` interface
- **CORS Enabled**: Configured for Vue frontend

### Frontend (Vue 3)
- **Vue 3 Composition API**: Modern reactive framework
- **Component-Based**: Modular components for form and list views
- **Form Validation**: Real-time validation with error messages
- **Search Functionality**: Search across all submission fields
- **Responsive Design**: Modern CSS with gradient backgrounds

## Getting Started

### Prerequisites

- .NET 10 SDK
- Node.js 18+ and npm

### Installation

1. **Install Node.js dependencies** (for frontend):
   ```bash
   cd Client
   npm install
   ```

2. **Build the frontend**:
   ```bash
   npm run build
   ```
   
   This will bundle all JavaScript and CSS files into `Server/wwwroot/`.

3. **Install .NET dependencies** (if needed):
   ```bash
   cd ../Server
   dotnet restore
   ```

### Running the Application

1. **Build the frontend** (if not already built):
   ```bash
   cd Client
   npm run build
   ```

2. **Run the .NET application**:
   ```bash
   cd ../Server
   dotnet run
   ```

3. Open your browser and navigate to:
   - `https://localhost:7064` or `http://localhost:5058` (depending on your launch settings)

### Development Mode

For development with hot-reload:

1. **Terminal 1 - Run the backend**:
   ```bash
   cd Server
   dotnet run
   ```

2. **Terminal 2 - Run the frontend dev server**:
   ```bash
   cd Client
   npm run dev
   ```

   The Vite dev server will run on `http://localhost:5173` and proxy API requests to the backend.

## API Endpoints

### POST /api/submissions
Submit a new form.

**Request Body**: Any JSON object (dynamic structure)
```json
{
  "name": "John Doe",
  "email": "john@example.com",
  "subject": "general",
  "preferredContact": "email",
  "date": "2024-12-31",
  "newsletter": true,
  "terms": true
}
```

**Response**: 
```json
{
  "id": "guid-here"
}
```

### GET /api/submissions
Get all submissions.

**Query Parameters**:
- `search` (optional): Search term to filter submissions

**Response**: Array of submission objects

### GET /api/submissions/{id}
Get a specific submission by ID.

**Response**: Submission object

## Form Fields

The contact form includes:
- **Text**: Full Name, Email Address
- **Dropdown**: Subject (General Inquiry, Technical Support, Sales Question, Feedback)
- **Date**: Preferred Contact Date
- **Radio**: Preferred Contact Method (Email, Phone, Both)
- **Checkbox**: Newsletter subscription, Terms and conditions

## Validation Rules

- **Name**: Required, non-empty
- **Email**: Required, valid email format
- **Subject**: Required, must select from dropdown
- **Preferred Contact**: Required, must select one option
- **Date**: Required, cannot be in the past
- **Terms**: Required, must be checked

## Design Decisions

1. **Dynamic Data Handling**: The backend uses `Dictionary<string, object>` to accept any form structure, making it flexible for different frontend applications.

2. **In-Memory Storage**: Uses `ConcurrentDictionary` for thread-safe, in-memory storage. No database setup required.

3. **Service Layer**: Clean architecture with interface-based service layer for testability and maintainability.

4. **Vite Build System**: Modern, fast build tool that bundles all assets into optimized files.

5. **Component Architecture**: Vue components are modular and reusable.

6. **Responsive Design**: Modern CSS with gradients and smooth transitions for better UX.

7. **Separated Client/Server**: Clear separation between frontend (Client/) and backend (Server/) for better organization.

## Testing the Dynamic Nature

To test that the backend accepts any form structure, you can create a different form (e.g., an order form) and submit it to the same `/api/submissions` endpoint. The backend will store it without any code changes.

Example order form submission:
```json
{
  "productName": "Widget",
  "quantity": 5,
  "shippingMethod": "express",
  "paymentMethod": "credit",
  "deliveryDate": "2024-12-25",
  "giftWrap": true
}
```

All submissions will appear in the list view, regardless of their structure.
